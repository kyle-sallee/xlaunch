A desktop environment is not provided.
A window manager is not provided.
The code megabytes required for window manager fluff is not provided.

By XLaunch an X11 hot key program launcher is provided.
A few kilobytes required by window geometry modification code is provided.
A couple tricks not found in or rare among the window manager niche are provided.

Who does not like "try before buy?"
How about "try and not buy?"
Does that seem better?

XLaunch is free software.
Within the freedom XLaunch must remain.
To the paydom XLaunch must never be exported.

To use or not to use; that is the question.
By downloading a source tarball; less a fraction of a megabyte bandwidth is consumed.
Instead by watching the 90 second each youtube hosted demonstration videos hundreds of megabytes of bandwidth can be consumed. Why not commit 10 minutes and a hundreds of megabytes, right?
The URLs to youtube hosted demonstration videos follow.

Xlaunch requirements are minimal. Naturally, a POSIX, portable operating system for running UNIX like software, is required.
A Linux API translator probably does not suffice.

Software projects xorg and linux-pam are required.
If linux-pam is not installed
then program xl-lock only can not be compiled.
In the README file the required software, universal resource locators, and command lines are provided.

For configuration file editing; any text editor will suffice.

For allocating time to investigate this project; thanks.
Software developer information follows.

At release; for a few years already XLaunch has been in use.
A larger than mozilla firefox hot key program is not desired and was not created.
A window manager similar experience will not become.
Most of the window manager solicited irritations, and a few of the window manager provided conveniences are omitted.

Before contemplating an XLaunch fork; the library interface should be inspected.
By linking with the XLaunch library; by new software projects; XLaunch provided shared library functions and variable structures can be used.
In that fashion XLaunch is extensible.

To the XLaunch software suite for an additional program to be added an entire software fork is not required.
If the project is published separately and code from XLaunch is not copied then any license can be applied.

An example follows.
By the provided window modification code an optimal experience might not become. Larger than root windows can be created.
From the root window visible area any window portion can depart.

During modification a change in anchor point might be required.
If an immobile cursor was used
then an anchor point would not be required. A different experience would become.

Several window modification methods were tried.
Some were abandoned.
Ultimately, the software author's preferences are provided.
However, more than one good window size/slide method exists.
Due to the XLaunch design in lieu of any method by the same key combination or by a different key combination an entirely different program can be launched.

Perhaps curiosity concerning the used graphics toolkit library exists?
Is gnome's GTK used? Is kde's QT used? The answer is no.
The X11 library, probably already required by most X11 client software, is used.
And a few other xorg provided libraries are used.

The mesa library, nvidia provided pre-compiled libraries and the fancy screen saver hacks are not provided and are not required.
If desired then at youtube in the video description the software and home pages are cited.
The default configuration file and the during video recording configuration file are similar.
Therefore, by anyone the same tricks can be accomplished.
